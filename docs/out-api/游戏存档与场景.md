# 游戏存档与场景接口文档

## 来源

- Controller：`packages/api/src/game.controller.ts`
- Service：`packages/api/src/game-api.service.ts`
- 数据库：`packages/api/src/infrastructure/database.ts`
- 类型：`packages/core/src/host.ts`
- 前端调用：`packages/web/src/host/request-client.ts`

## 接口清单

| 方法 | 路径 | 用途 | 状态 |
|---|---|---|---|
| POST | `/game/save` | 保存当前游戏快照 | confirmed |
| GET | `/game/latest` | 读取最近一次游戏快照 | confirmed |
| POST | `/game/load` | 按 runId 读取游戏快照 | confirmed |
| POST | `/game/scenario/generate` | 根据参考小说生成开局场景 | confirmed |
| GET | `/game/saves` | 列出可读取的存档摘要 | confirmed |
| POST | `/game/delete` | 删除指定存档 | confirmed |
| POST | `/game/death-archives` | 保存死亡归档 | confirmed |
| POST | `/game/inventory-pins` | 保存背包置顶项 | confirmed |
| POST | `/game/inventory-pins/load` | 读取背包置顶项 | confirmed |

## 保存当前游戏

### 请求

`POST /game/save`

请求体为当前游戏快照对象，当前 Controller 类型为 `Record<string, unknown>`。

### 响应

```json
{
  "success": true,
  "message": "存档成功",
  "data": {},
  "runId": "run_..."
}
```

### 错误码

MISSING：当前没有字段级校验与稳定错误码。

### 联调说明

- 服务端会将快照写入 SQLite，并返回实际 `runId`。
- `data` 为落库后的快照内容。

## 读取最近存档

### 请求

`GET /game/latest`

### 参数

无。

### 响应

```json
{
  "success": true,
  "data": null
}
```

### 错误码

MISSING：当前没有字段级校验与稳定错误码。

### 联调说明

- 没有存档时 `data` 为 `null`，不是业务失败。

## 按 ID 读取存档

### 请求

`POST /game/load`

| 名称 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| runId | body | string | 是 | 存档 ID |

### 响应

```json
{
  "success": true,
  "data": {}
}
```

### 错误码

MISSING：当前没有字段级校验与稳定错误码。

### 联调说明

- runId 不存在时 `data` 为 `null`。

## 生成开局场景

### 请求

`POST /game/scenario/generate`

| 名称 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| referenceNovel | body | string | 是 | 参考小说名 |
| author | body | string | 否 | 作者 |
| description | body | string | 否 | 小说摘要 |
| difficulty | body | string | 是 | 难度 |
| gameTypeId | body | string | 是 | 游戏类型 |
| themeId | body | string | 是 | 主题 |

### 响应

```json
{
  "success": true,
  "data": {}
}
```

失败时可能返回：

```json
{
  "success": false,
  "message": "失败原因"
}
```

### 错误码

当前接口没有稳定错误码；失败原因通过 `message` 返回。

### 联调说明

- 场景生成由 `generateScenarioWithAI` 负责，依赖当前运行时 AI 配置。
- `data` 结构由场景生成逻辑返回，MISSING：当前没有公开 DTO。

## 列出存档

### 请求

`GET /game/saves`

### 参数

无。

### 响应

```json
{
  "success": true,
  "data": [
    {
      "runId": "run_...",
      "playerName": "角色名",
      "realm": "境界",
      "currentScene": "场景",
      "updatedAt": "2026-06-07T00:00:00.000Z"
    }
  ]
}
```

### 错误码

MISSING：当前没有字段级校验与稳定错误码。

### 联调说明

- 返回按数据库实现排序的存档摘要数组。
- 摘要可包含 `gameTypeId`、`themeId`、`referenceNovel`。

## 删除存档

### 请求

`POST /game/delete`

| 名称 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| runId | body | string | 是 | 待删除存档 ID |

### 响应

```json
{
  "success": true,
  "message": "存档已删除",
  "runId": "run_..."
}
```

不存在时返回：

```json
{
  "success": false,
  "message": "存档不存在或已删除"
}
```

### 错误码

当前接口没有稳定错误码。

### 联调说明

- 删除失败不使用非 2xx HTTP 状态。
