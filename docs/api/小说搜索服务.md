# 小说搜索服务接口文档

## 来源

- 消费入口：`packages/api/src/novel-search.ts`
- 默认配置：`packages/core/src/novel-api-config.ts`
- 设置页：`packages/web/src/components/game/src/settings-novel-api-tab.tsx`

## 接口清单

| 方法 | 路径 | 用途 | 状态 |
|---|---|---|---|
| POST | `<novelApiBaseURL>/novels/search` | 兼容协议小说搜索 | confirmed |
| GET/POST | 自定义函数返回的 `url` | 用户脚本适配远程小说搜索 | confirmed |

## 兼容协议小说搜索

### 请求

`POST <novelApiBaseURL>/novels/search`

| 名称 | 位置 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| Content-Type | header | string | 是 | `application/json` |
| Authorization | header | string | 否 | 配置 `novelApiKey` 时发送 `Bearer <key>` |
| X-API-Key | header | string | 否 | 配置 `novelApiKey` 时发送 |
| keyword | body | string | 是 | 搜索关键词 |

### 响应

```json
{
  "success": true,
  "data": [
    {
      "id": "novel-id",
      "title": "小说名",
      "author": "作者",
      "description": "摘要",
      "source": "来源"
    }
  ],
  "message": "可选说明"
}
```

### 错误码

外部服务错误未映射为稳定错误码；HTTP 非 2xx 返回 `小说兼容 API 请求失败：HTTP <status>`。

### 联调说明

- 服务端使用 Zod 校验响应必须包含 `success: boolean` 与小说摘要数组。
- Base URL 缺失时不会发起远程请求，直接返回 `小说 API Base URL 未配置`。

## 自定义函数适配

### 请求

请求由用户配置的 `novelApiBuildRequestCode` 生成，返回结构必须符合：

```json
{
  "url": "https://example.com/search?q=keyword",
  "method": "GET",
  "headers": {},
  "body": {}
}
```

### 响应

远程响应会先作为 JSON 传给 `novelApiMapResponseCode`，映射函数必须返回：

```json
{
  "novels": [
    {
      "id": "novel-id",
      "title": "小说名",
      "author": "作者",
      "description": "摘要",
      "source": "来源"
    }
  ],
  "message": "可选说明"
}
```

### 错误码

外部服务错误、函数执行错误或 schema 校验失败统一写入 `message`。

### 联调说明

- 自定义函数在 `node:vm` 的新上下文执行，超时时间为 1000ms。
- POST 请求会自动补充 `Content-Type: application/json` 并 JSON 序列化 `body`。
- 默认函数示例使用 Open Library 搜索接口；默认 provider 已切到 `ai`，Open Library 只作为可配置示例。
